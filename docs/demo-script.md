# Demo Script

Personal reference for the live walkthrough — not a polished deliverable, just talking points and a
suggested click-path. Read from this, don't read it aloud verbatim.

---

## 1. The pitch (30 seconds)

> "This is an AI-native board reporting platform, built for a graduate assessment set by Assiduous
> Corp. The brief: take a real company's financial filings and turn them into a board report —
> Management, the Board, investors, and credit providers all need the same numbers presented
> differently. I used a real company as the case study — Senus PLC, an Irish natural-capital SaaS
> company listed on Euronext Access Dublin — and every number on this dashboard comes from their
> actual public filings, not synthetic data."

## 2. My approach, in one breath (1 minute)

> "The core design decision was: **deterministic extraction first, AI second.** Most people would
> reach for an LLM to read a PDF and just... trust whatever it says. I didn't want to build that,
> because it's not auditable and it hallucinates. Instead, I built a real regex/table parser that
> reads the filing's actual structure first — the P&L, the balance sheet, the cash flow statement —
> and only falls back to Gemini when the deterministic pass genuinely can't find something, or when
> the PDF is a scan with no text layer at all. And every single extraction gets a transparent
> confidence score — not a made-up AI 'confidence,' a real point system based on *how* each number was
> found. Below a threshold, it never reaches the dashboard until a human confirms it."

Optional follow-up line if asked "why does that matter":
> "Because the first time I got this wrong — a non-financial document got ingested and silently
> zeroed out real data on the dashboard — that's exactly the failure mode a board report can't afford.
> The confidence system exists because that actually happened once, and I built the fix so it
> structurally can't happen again, not just patched for that one case."

## 3. Live walkthrough

### 3a. Executive Dashboard (`/`)

- Point at the **hero KPI row**: Total Revenue, EBITDA, Cash Position, Active Customers. Note the
  **period banner** ("All figures in EUR · Data as of...") and the **period selector** top-right.
- Click the period selector, switch between **FY2025** and **HY2026** — the whole dashboard re-anchors:
  KPI cards, the stat strip below, AI insights. Say: "Senus has two filings on different reporting
  cadences — a half-year and a full year — and the system never blends them into one misleading trend.
  Picking a period shows that period's own real comparison, not a fabricated one."
- Point at the **secondary stat strip**: Bookings, EBITDA Margin, Cash Runway, Interest Cover, ROCE —
  "these are the five assignment-required categories: Growth & Revenue, Profitability, Cash &
  Liquidity, Solvency & Leverage, Returns."
- Point at a **negative value** (EBITDA, or a margin) rendering in red even with no visible trend arrow
  — "the color reflects the number itself being bad, not just whether it moved."
- **AI Board Insights panel**: "Three insights, each tagged Positive/Risk/Opportunity, each with a
  recommended board action — generated by Gemini from the actual computed KPIs, not the raw PDF." Point
  at the refresh button — "this is quota-aware: it won't re-spend API calls on data that hasn't
  changed, and it only marks itself 'done' after a genuinely successful generation, not a fallback."
- **Revenue Trend chart**: toggle Revenue / EBITDA / Cash. Toggle **Show forecast** — "a simple linear
  projection, visually distinct (dashed, indigo, italic labels) so it's never mistaken for a real
  historical point."

### 3b. Documents page (`/documents`)

- Show the **Uploaded Documents** table. Point out the **status badges**: "Pending Review" (a document
  that scored 85–94% confidence — real data, but not yet trusted for the dashboard), "Rejected" (below
  85%, kept for reference, can never reach the dashboard), "Merged" (two documents found to report the
  *same* period, combined into one clean record).
- Click the **Review** icon on a Pending Review or Rejected document — show the panel: extracted
  values, the confidence score, and **"Why this score"** — the actual point breakdown, e.g. "Revenue
  not found (0/30)." For a needs_review one, click **Approve for dashboard** — "a human confirms it's
  correct, and that unlocks it for the dashboard — but the underlying score is never rewritten, so the
  system's own history of what it actually found stays honest."
- Point at the **investor relations banner** at the top ("N new filings available") — "this pulls
  directly from Senus's own investor relations API, not a scrape — I reverse-engineered the real
  underlying JSON endpoints." Click **Check now**. If there's anything hideable, show the **eye-off**
  icon — "mark a filing as out of scope if it turns out to have no financial data, without losing
  track that it was reviewed."
- **Upload a PDF** live if time allows — walk through what happens: extraction → confidence scoring →
  either lands on the dashboard immediately, or shows up tagged for review.

### 3c. Reports page (`/reports`)

- Quick pass: search/filter, status badges, CSV export.

### 3d. Settings (`/settings`)

- Toggle **Light / Dark / System** theme — "a small thing, but dark mode used to be completely dead
  code here — dozens of classes with nothing ever applying them. Fixed properly, not just cosmetically."

## 4. If there's time: architecture highlights

- **Extraction confidence service** (`extraction_confidence.py`) — the point system, the two
  arithmetic reconciliation checks (does revenue − cost of sales = gross profit; does the cash flow
  statement's own components sum to the stated net change).
- **Vision extraction fallback** — one of Senus's real filings (ADF Farm Solutions' predecessor-entity
  statements) is a scanned PDF with no text layer at all. Built a Gemini *vision* backup specifically
  gated to only trigger when the deterministic pipeline structurally cannot read a document — never
  used for one it could already read.
- **Same-period document merge** — two of Senus's real filings turned out to report the *exact same*
  underlying period from two different source documents. Rather than picking one arbitrarily, the
  system merges them field-by-field, and flags any genuine disagreement for human review instead of
  silently choosing.
- **Architecture Decisions table in the README** — a scannable log of every real design choice made
  and the alternative that was on the table, if asked "why did you build it this way."

## 5. Honest close

> "A few things I'm deliberately not claiming are done: PDF storage on the hosting platform is
> ephemeral right now, so an uploaded file can be lost across a redeploy — the extracted data survives
> fine, just not the original PDF, and that's a known, documented limitation, not a hidden bug. There's
> no real authentication, because this is a single-user boardroom tool by design, not a multi-tenant
> product. And the AI Board Insights panel's own Gemini quota can run out — when it does, it falls
> back to clearly-static content rather than pretending, and I built real backoff logic so it doesn't
> keep wasting calls against an exhausted quota."

---

## Quick reference: what to click, in order

1. `/` — period selector → FY2025 / HY2026, forecast toggle, Revenue/EBITDA/Cash toggle
2. `/documents` — Pending Review → Review panel → Approve; Rejected → Review panel (view-only); Merged
   tag; investor relations banner; upload
3. `/reports` — search/filter, CSV export
4. `/settings` — theme toggle
5. Back to `/` to close on the dashboard
